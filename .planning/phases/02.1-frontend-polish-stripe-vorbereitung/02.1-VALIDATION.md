# Phase 02.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Befüllt nach RESEARCH.md "Validation Architecture"-Empfehlungen.
> Konkrete Task-Mappings ergänzt der Planner pro Plan-Datei.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | vitest (bestehend, siehe `vitest.config.ts`) + Stripe-CLI für Webhook-Smoke |
| **Config file** | `vitest.config.ts` |
| **Quick run command** | `npm run test -- --run` (Vitest in single-run-Mode, kein Watch) |
| **Full suite command** | `npm run test -- --run && npm run typecheck && npm run lint` |
| **Estimated runtime** | ~30 s Quick / ~90 s Full / Stripe-Webhook-Smoke ~10 s separat |

---

## Sampling Rate

- **After every task commit:** Run `npm run test -- --run` (Vitest single-run, kein Watch-Mode)
- **After every plan wave:** Run full suite (`test --run` + `typecheck` + `lint`)
- **Before `/gsd-verify-work`:** Full suite + Stripe-CLI-Smoke + Reload-Resume-Smoke + visueller Mobile-Audit-Bericht müssen grün
- **Max feedback latency:** 30 s
