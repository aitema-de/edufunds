## Deviationen vom Plan

### Auto-fix: Mock-Strategy fuer Tests angepasst (Rule 1 — Bug)

**Gefunden waehrend:** Task 1 (pipeline-events) + Task 3 (WizardShell)

**Problem 1:** Plan-Vorschlag nutzte `{ data: {...} }` als generateJson-Mock-Return — tatsaechliches Interface ist `{ value: T, usage: Usage }`. Ausserdem konnten Inline-Konstanten nicht im jest.mock()-Factory-Callback referenziert werden (Hoisting-ReferenceError).

**Fix:** Mock-Werte inline im Factory-Callback definiert, `value`-Schlussel korrekt gesetzt, STUB_JSON_VALUE mit allen Pipeline-Stage-Shapes (outline + critique + finanzplan) zusammengefasst.

**Problem 2:** `jest.spyOn(global, 'fetch')` wirft "Property fetch does not exist" in jsdom; `new Response(...)` wirft "Response is not defined".

**Fix:** `global.fetch = jest.fn()` direkt zugewiesen (save/restore in before/afterEach); `mockResponse()`-Hilfsfunktion mit plain Object `{ ok, status, json(), text() }` statt `new Response()`.

**Problem 3:** WizardShell importiert AntragResult, das transitiv react-markdown importiert (ESM-only, nicht durch Jest transformierbar).

**Fix:** `jest.mock('@/components/Wizard/AntragResult', () => ({ AntragResult: () => null }))` hinzugefuegt.
